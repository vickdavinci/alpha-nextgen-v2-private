### V12.19 Errata — RETRY_CANCELED_CLOSE Mistagging Fix

**Bug:** In V12.17 logs, all RETRY_CANCELED_CLOSE exit orders were tagged `MICRO:RETRY_CANCELED_CLOSE` regardless of the actual engine. The 10 ITM trades that exited via RETRY_CANCELED (T1, T4, T6, T11, T13, T18, T21, T23, T25, T28) have correct `ITM:ITM_MOMENTUM` **entry** tags but incorrect `MICRO:RETRY_CANCELED_CLOSE` **exit** tags in the log.

**Root cause:** The `ExitOrderTracker` did not preserve engine attribution at tracker creation time. When the retry path fired, `_infer_option_engine_bucket_for_symbol` received `tag_hint="CANCELED_CLOSE"` (no engine keyword) and defaulted to "MICRO".

**Fix (V12.19):** Added `engine_bucket` field to `ExitOrderTracker`. Engine bucket is now computed and stored at tracker creation (both cancel and reject handlers), then passed as `engine_hint` to `_submit_option_close_market_order` during retry. Future backtests will correctly tag retry exits.

**Impact on this report:** Trade classification in this report was done by cross-referencing **entry** fill tags (which were correct), so all ITM/MICRO trade attribution in the tables below is accurate despite the exit tag bug.
