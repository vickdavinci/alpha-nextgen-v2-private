MODIFICATION 3: ADD OCO ORDER MANAGEMENT (SECTION 8 ORDER EXECUTION)
═══════════════════════════════════════════════════════════════════════════════

PROBLEM: "Limit Order Race Condition"

  Current Implementation (WRONG):
    1. Entry fills at $1.42
    2. Place limit sell at $2.13 (profit target)  ← GTC order 1
    3. Place limit sell at $1.065 (stop loss)    ← GTC order 2

  Risk Scenarios:
    A) Price whipsaws to $2.13, then back to $1.065
       Both orders could execute (impossible, but race condition)

    B) Price drops to $1.065, stop fills
       Profit target order still floating (Ghost order)
       Must manually cancel it (error-prone)

    C) Price spikes to $2.13, target fills
       Stop order still active below (Ghost order)
       Creates unexpected position if price drops again

SOLUTION: USE OCO (ONE-CANCELS-OTHER) ORDERS

OCO Mechanics:
  - Two linked orders (profit target & stop loss)
  - Only ONE can fill
  - When one fills → other cancels automatically
  - No manual intervention needed
  - Supported by Interactive Brokers via QuantConnect
