ENGINE 1: TREND ENGINE (V2.1 SPECIFICATION)
────────────────────────────────────────────────────────────────────────────

PURPOSE:
  Capture long-term trend on leveraged ETFs (QLD 3x or SSO 3x Nasdaq/S&P)

ALLOCATION: 70% of portfolio capital

TIMEFRAME: Daily or swing (3-30 day holds typical)

INSTRUMENTS:
  Primary: QLD (Invesco QQQ 3x daily ETF) or SSO (ProShares 3x S&P 500)
  Why: Leverage captures trends efficiently, lower slippage than futures

2.1.1 MATHEMATICAL FOUNDATION

Core Principle: CONFIDENCE-WEIGHTED POSITION SIZING

  Fixed Risk Model: Each trade risks exactly 1% of portfolio

  Formula:
    Risk_$ = Portfolio_Value × 0.01 = constant (e.g., $1,000 on $100k)

    Contracts = Risk_$ / (Stop_Loss_Pct × Entry_Premium × 100)

    Therefore: Higher confidence (wider stop) → Smaller position
              Lower confidence (tighter stop) → Larger position

    Both maintain exactly 1% portfolio risk ✓

2.1.2 ENTRY SIGNAL: MA200 + ADX CONFIRMATION

STEP 1: MA(200) Baseline Trend

  Calculation:
    ma200 = SMA(close, 200)
    signal = "BUY" if close > ma200 else "HOLD/SELL"

  Interpretation:
    Close > MA200 → Price above long-term average → Bullish
    Close < MA200 → Price below long-term average → Bearish

  Timing:
    - Calculate daily after market close (4:00 PM ET)
    - Check signal on next bar's open

  Crossover Detection:
    previous_ma200 = ma200_from_yesterday
    current_ma200 = ma200_today

    IF previous_close < previous_ma200 AND current_close > current_ma200:
      → Golden Cross detected (bullish)
      → ENTRY TRIGGER (if ADX confirms)

    IF previous_close > previous_ma200 AND current_close < current_ma200:
      → Death Cross detected (bearish)
      → EXIT TRIGGER (if in long position)

STEP 2: ADX(14) Momentum Confirmation

  Calculation (14-period ADX):
    True Range = max(high - low, |high - prev_close|, |low - prev_close|)
    +DM = high - prev_high (if positive and > -DM, else 0)
    -DM = prev_low - low (if positive and > +DM, else 0)

    +DI = 100 × SMA(+DM, 14) / SMA(TR, 14)
    -DI = 100 × SMA(-DM, 14) / SMA(TR, 14)

    ADX = 100 × SMA(|+DI - -DI| / (+DI + -DI|), 14)  [0-100 scale]

  Interpretation:
    ADX > 35: Very strong trend (BEST confidence)
    ADX 25-35: Strong trend (HIGH confidence)
    ADX 20-25: Moderate trend (MEDIUM confidence)
    ADX < 20: Weak/choppy (LOW confidence, possibly exit)

  Scoring:
    score_adx = 0.25  if ADX < 20
    score_adx = 0.50  if 20 <= ADX < 25
    score_adx = 0.75  if 25 <= ADX < 35
    score_adx = 1.0   if ADX >= 35

  Logic:
    IF ma200_signal == "BUY" AND score_adx >= 0.50:
      → Proceed to Kelly sizing
    ELSE:
      → HOLD (wait for stronger ADX confirmation)

STEP 3: KELLY CRITERION SIZING

  Kelly Formula:
    kelly_pct = (win_rate × avg_win - loss_rate × avg_loss) / avg_win

    Where:
      win_rate = wins / total_trades (from state_manager)
      avg_win = average % gain on winners
      avg_loss = average % loss on losers
      loss_rate = 1 - win_rate

  Fractional Kelly:
    kelly_frac = kelly_pct × 0.5  (use 50% of full Kelly for safety)

  Example Calculation:
    Historical performance: 50 trades, 25 wins, 25 losses
    win_rate = 25/50 = 0.50
    avg_win = +2.5% on winners
    avg_loss = -2.5% on losers

    Full kelly = (0.50 × 2.5 - 0.50 × 2.5) / 2.5 = 0 / 2.5 = 0
    → Breakeven, position = 0 (too risky)

    Better example (ADX filter improves): 50 trades, 28 wins, 22 losses
    win_rate = 28/50 = 0.56
    avg_win = +3.0%
    avg_loss = -2.5%

    Full kelly = (0.56 × 3.0 - 0.44 × 2.5) / 3.0 = (1.68 - 1.10) / 3.0 = 0.193
    Frac kelly = 0.193 × 0.5 = 0.0965 ≈ 9.65% leverage
    → Position = leverage × 1x base = ~1.1x (minimal leverage)

  Constraint:
    kelly_frac must be >= 0 and <= 0.50 (never exceed 50% Kelly)
    If kelly_frac < 0 → Skip trading entirely (system is losing)
    If kelly_frac > 0.50 → Cap at 0.50 (safety limit)

STEP 4: POSITION SIZING WITH VOLATILITY TARGETING

  Realized Volatility Calculation:
    recent_returns = log(close / prev_close) for last 50 days
    realized_vol = STDEV(recent_returns) × sqrt(252)

    Example: realized_vol = 0.15 (15% annualized)

  Daily Vol from Realized:
    daily_vol = realized_vol / sqrt(252) ≈ realized_vol / 15.87

    Example: daily_vol = 0.15 / 15.87 ≈ 0.0094 (0.94% daily)

  Vol Targeting:
    target_vol = 0.01  (1.0% daily portfolio volatility target)

    vol_scalar = target_vol / daily_vol

    Example: vol_scalar = 0.01 / 0.0094 ≈ 1.06 (1.06x scaling)

  Position Size Calculation:
    base_leverage = kelly_frac × vol_scalar

    position_size = base_leverage × 1x
    → How many "units" of the ETF to hold

  Example:
    kelly_frac = 0.10
    vol_scalar = 1.06
    base_leverage = 0.10 × 1.06 = 0.106

    On $100k portfolio: position_size = 0.106 × $100k = $10,600
    → Buy ~$10,600 of QLD (at 3x leverage, controls $31,800 notional)

STEP 5: STOP LOSS AND PROFIT TARGET

  Stop Loss:
    Fixed at -2% from entry
    stop_price = entry_price × 0.98

    Example: Entry at $450 → Stop at $441
    max_loss_per_trade = position_size × 0.02 = 1% of portfolio (approx)
    → Aligns with risk model

  Profit Target:
    Trailing or fixed at +5%
    target_price = entry_price × 1.05

    Example: Entry at $450 → Target at $472.50

  Exit Logic:
    Close position if:
      • Stop hit (loss management)
      • Target hit (profit taking)
      • MA200 crossover below (trend break)
      • ADX drops below 20 (trend weakness)
      • Daily close below MA200 (end of bar)

```python
class TrendEngine:
    def __init__(self, portfolio_value, kelly_lookback=50):
        self.portfolio_value = portfolio_value
        self.kelly_lookback = kelly_lookback
        self.position = None
        self.entry_price = None

    def update(self, price_data, trade_history):
        # Calculate MA200
        ma200 = ta.SMA(price_data['close'], 200)
        ma_signal = "BUY" if price_data['close'][-1] > ma200[-1] else "SELL"

        # Calculate ADX
        adx = ta.ADX(price_data['high'], price_data['low'], 
                     price_data['close'], 14)
        score_adx = self._score_adx(adx[-1])

        # Check entry conditions
        if ma_signal == "BUY" and score_adx >= 0.50:
            # Calculate Kelly sizing
            kelly_pct = self._calculate_kelly(trade_history[-self.kelly_lookback:])
            kelly_frac = kelly_pct * 0.5

            # Calculate vol scalar
            realized_vol = self._calculate_realized_vol(price_data['close'][-50:])
            daily_vol = realized_vol / math.sqrt(252)
            vol_scalar = 0.01 / daily_vol

            # Calculate position size
            base_leverage = kelly_frac * vol_scalar
            position_size = base_leverage * self.portfolio_value

            # Store entry
            self.entry_price = price_data['close'][-1]
            self.position = {
                'size': position_size,
                'entry_price': self.entry_price,
                'stop_price': self.entry_price * 0.98,
                'target_price': self.entry_price * 1.05,
                'entry_time': price_data['timestamp'][-1]
            }

            return {
                'signal': 'BUY',
                'position_size': position_size,
                'leverage': base_leverage,
                'kelly': kelly_frac,
                'adx': adx[-1],
                'confidence': score_adx
            }

        # Check exit conditions
        if self.position:
            if price_data['close'][-1] <= self.position['stop_price']:
                self._exit_position(price_data['close'][-1], 'STOP_HIT')
            elif price_data['close'][-1] >= self.position['target_price']:
                self._exit_position(price_data['close'][-1], 'TARGET_HIT')
            elif ma_signal == "SELL" and score_adx < 0.50:
                self._exit_position(price_data['close'][-1], 'MA_CROSS_EXIT')

        return None

    def _score_adx(self, adx_value):
        if adx_value >= 35:
            return 1.0
        elif adx_value >= 25:
            return 0.75
        elif adx_value >= 20:
            return 0.50
        else:
            return 0.25

    def _calculate_kelly(self, trades):
        if len(trades) < 10:
            return 0  # Not enough history

        wins = [t for t in trades if t['pnl'] > 0]
        losses = [t for t in trades if t['pnl'] < 0]

        if not wins or not losses:
            return 0  # Can't calculate

        win_rate = len(wins) / len(trades)
        avg_win = sum([t['pnl_pct'] for t in wins]) / len(wins)
        avg_loss = -sum([t['pnl_pct'] for t in losses]) / len(losses)

        kelly_pct = (win_rate * avg_win - (1 - win_rate) * avg_loss) / avg_win

        return max(0, min(kelly_pct, 0.50))  # Cap at 50%

    def _calculate_realized_vol(self, closes):
        returns = [math.log(closes[i] / closes[i-1]) for i in range(1, len(closes))]
        return np.std(returns) * math.sqrt(252)

    def _exit_position(self, exit_price, reason):
        pnl_pct = (exit_price - self.entry_price) / self.entry_price
        self.position['exit_price'] = exit_price
        self.position['exit_reason'] = reason
        self.position['pnl_pct'] = pnl_pct
        # Log to state_manager for Kelly recalculation
        self.position = None
```

ENGINE 2: MEAN REVERSION ENGINE (V2.1 SPECIFICATION)
────────────────────────────────────────────────────────────────────────────

PURPOSE:
  Capture oversold/overbought bounces on index ETFs
  Counter-trend to Trend Engine (provides diversification)

ALLOCATION: 0-10% of portfolio (optional, activates on extreme moves)

TIMEFRAME: 1-5 day bounces (typically)

INSTRUMENTS:
  SSO/QLD (same as Trend, or QQQ for correlation isolation)

2.2.1 MATHEMATICAL FOUNDATION

Core Concept: MEAN REVERSION vs TREND FOLLOWING

  Trend Engine: Follows momentum
    Buy strength → Ride the wave → Sell on slowdown

  Mean Reversion Engine: Fights extremes
    Sell strength (oversold) → Bounce → Take quick profit
    Buy weakness (overbought) → Bounce → Take quick profit

  Mathematical Difference:
    Trend: Correlates with recent price direction (positive auto-correlation)
    MR: Correlates against recent price direction (negative auto-correlation)

  When to use MR:
    • Extreme down days (SPY -3%+ from open)
    • Extreme up days (SPY +3%+ from open)
    • Support/resistance bounces
    • Deviation from moving averages > 2 standard deviations

2.2.2 ENTRY SIGNAL: SUPPORT/RESISTANCE + RSI CONFIRMATION

STEP 1: Support/Resistance Detection

  Moving Support (Lower Bollinger Band):
    sma20 = SMA(close, 20)
    std20 = STDEV(close, 20)
    lower_bb = sma20 - (2 × std20)

    → Price touching lower_bb = potential support

  Historical Support Levels:
    recent_lows = [min(close[i:i+20]) for each window]
    support_levels = cluster_pivots(recent_lows)

    → Price bouncing from support = short-term bounce likely

  Resistance:
    upper_bb = sma20 + (2 × std20)
    recent_highs = [max(close[i:i+20]) for each window]
    resistance_levels = cluster_pivots(recent_highs)

STEP 2: RSI(14) Oversold/Overbought Detection

  RSI Calculation:
    gains = [close[i] - close[i-1] for i in 1..14 if positive]
    losses = [close[i-1] - close[i] for i in 1..14 if negative]

    avg_gain = SMA(gains, 14)
    avg_loss = SMA(losses, 14)

    rs = avg_gain / avg_loss
    rsi = 100 - (100 / (1 + rs))

    Range: [0-100]

  Signal Levels:
    RSI > 70: Overbought (potential short/sell signal)
    RSI < 30: Oversold (potential long/buy signal)
    RSI 30-70: Neutral zone

  Scoring:
    score_rsi = 0.0 if RSI > 60
    score_rsi = 0.75 if 50 <= RSI <= 60
    score_rsi = 1.0 if RSI < 30 (strong oversold)

    score_rsi = 0.0 if RSI < 40
    score_rsi = 0.75 if 40 <= RSI <= 50
    score_rsi = 1.0 if RSI > 70 (strong overbought)

STEP 3: ENTRY CONFIRMATION

  Long Entry (Bounce from oversold):
    Conditions:
      ✓ RSI < 30 (oversold)
      ✓ Price touches lower Bollinger Band (2 std below)
      ✓ Volume increasing (confirmation)
      ✓ NOT during trend engine long (avoid collision)

    Action: Buy at support + 0.5% (limit order)
    Thesis: Extreme selling exhaustion → technical bounce

  Short Entry (Bounce from overbought):
    Conditions:
      ✓ RSI > 70 (overbought)
      ✓ Price touches upper Bollinger Band (2 std above)
      ✓ Volume increasing (confirmation)
      ✓ NOT during trend engine short

    Action: Sell at resistance - 0.5% (limit order)
    Thesis: Extreme buying exhaustion → technical bounce

STEP 4: POSITION SIZING

  Risk Model: Same as Trend Engine (1% per trade)

  Calculation:
    stop_loss_pct = 0.04  (4% stop for MR, wider than trend 2%)
    → Bounces are less reliable, need wider stop

    risk_per_contract = entry_price × stop_loss_pct × 100
    position_size = $1,000 / risk_per_contract

    Example:
      Entry: $450
      Stop: $450 × 0.96 = $432
      Risk: $450 × 0.04 × 100 = $1,800 per 100 shares
      Position: $1,000 / $18 = 55 shares ≈ $24,750 notional (acceptable risk)

  Constraint:
    Max allocation to MR: 10% of capital
    If condition triggers, can only trade if:
      • Trend position is small or flat
      • Options position is not maximum
      • Correlation is low (< 0.50)

STEP 5: PROFIT TARGET AND STOP LOSS

  Profit Target:
    Short-term bounce target = +1.5% to +2.5%

    For long bounces: entry_price × 1.02
    For short bounces: entry_price × 0.98

    Example: Entry at $450 → Target at $459

  Stop Loss:
    Wider than Trend (4% vs 2%) because:
    • Bounces are less reliable
    • Need room for false breakdowns
    • Mean reversion takes longer

    For long bounces: entry_price × 0.96
    For short bounces: entry_price × 1.04

    Example: Entry at $450 → Stop at $432

STEP 6: TIME STOP

  Hold Duration:
    max_hold = 5 days
    If target not hit within 5 days → Exit (close at market)

    Rationale: MR thesis is short-term
    If bounce hasn't worked in 5 days, thesis is broken

2.2.3 IMPLEMENTATION PSEUDOCODE

```python
class MeanReversionEngine:
    def __init__(self, portfolio_value, max_allocation=0.10):
        self.portfolio_value = portfolio_value
        self.max_allocation = max_allocation
        self.position = None
        self.entry_price = None

    def update(self, price_data, trend_position, correlation):
        # Only trade if Trend is small and correlation is low
        if trend_position and trend_position['size'] > 0.05 * self.portfolio_value:
            return None  # Skip if Trend is dominant

        if correlation > 0.50:
            return None  # Skip if correlated

        # Calculate support/resistance
        sma20 = ta.SMA(price_data['close'], 20)
        std20 = ta.STDEV(price_data['close'], 20)
        lower_bb = sma20[-1] - (2 * std20[-1])
        upper_bb = sma20[-1] + (2 * std20[-1])

        # Calculate RSI
        rsi = ta.RSI(price_data['close'], 14)
        rsi_value = rsi[-1]

        current_price = price_data['close'][-1]

        # Long Entry (Oversold bounce)
        if rsi_value < 30 and current_price <= lower_bb:
            entry_price = lower_bb + (current_price - lower_bb) * 0.005
            stop_price = entry_price * 0.96
            target_price = entry_price * 1.02

            position_size = min(
                (self.portfolio_value * 0.01) / ((entry_price - stop_price) * 100),
                self.portfolio_value * self.max_allocation / entry_price
            )

            self.position = {
                'type': 'LONG',
                'size': position_size,
                'entry_price': entry_price,
                'stop_price': stop_price,
                'target_price': target_price,
                'entry_time': price_data['timestamp'][-1],
                'rsi': rsi_value,
                'support_level': lower_bb
            }

            return {
                'signal': 'LONG_BOUNCE',
                'position_size': position_size,
                'rsi': rsi_value,
                'support': lower_bb
            }

        # Short Entry (Overbought bounce)
        if rsi_value > 70 and current_price >= upper_bb:
            entry_price = upper_bb - (upper_bb - current_price) * 0.005
            stop_price = entry_price * 1.04
            target_price = entry_price * 0.98

            position_size = min(
                (self.portfolio_value * 0.01) / ((stop_price - entry_price) * 100),
                self.portfolio_value * self.max_allocation / entry_price
            )

            self.position = {
                'type': 'SHORT',
                'size': position_size,
                'entry_price': entry_price,
                'stop_price': stop_price,
                'target_price': target_price,
                'entry_time': price_data['timestamp'][-1],
                'rsi': rsi_value,
                'resistance_level': upper_bb
            }

            return {
                'signal': 'SHORT_BOUNCE',
                'position_size': position_size,
                'rsi': rsi_value,
                'resistance': upper_bb
            }

        # Exit conditions
        if self.position:
            if self.position['type'] == 'LONG':
                if current_price <= self.position['stop_price']:
                    self._exit_position(current_price, 'STOP_HIT')
                elif current_price >= self.position['target_price']:
                    self._exit_position(current_price, 'TARGET_HIT')

            elif self.position['type'] == 'SHORT':
                if current_price >= self.position['stop_price']:
                    self._exit_position(current_price, 'STOP_HIT')
                elif current_price <= self.position['target_price']:
                    self._exit_position(current_price, 'TARGET_HIT')

            # Time stop (5 days)
            days_held = (price_data['timestamp'][-1] - self.position['entry_time']).days
            if days_held >= 5:
                self._exit_position(current_price, 'TIME_STOP')

        return None

    def _exit_position(self, exit_price, reason):
        if self.position['type'] == 'LONG':
            pnl_pct = (exit_price - self.position['entry_price']) / self.position['entry_price']
        else:
            pnl_pct = (self.position['entry_price'] - exit_price) / self.position['entry_price']

        self.position['exit_price'] = exit_price
        self.position['exit_reason'] = reason
        self.position['pnl_pct'] = pnl_pct
        self.position = None
```

```python
    # Filter by DTE
    eligible = [c for c in available_contracts if 1 <= c.dte <= 4]

        eligible = [c for c in eligible if 0.40 <= c.delta <= 0.60]

    candidates = sorted(eligible,
```

2.3.8 ORDER EXECUTION WITH OCO (V2.1 MODIFICATION #3)

CRITICAL: Use One-Cancels-Other (OCO) order pairs to avoid ghost orders

WHY OCO IS MANDATORY:
  Two separate GTC orders create race condition
  If profit target fills, stop remains active (ghost order)
  If price drops below stop later, we get a second fill
  This creates negative position or unexpected loss

OCO PROCESS:

STEP 1: Place Entry Order
  Type: LimitOrder
  Price: ask - 5% (5% discount to improve fill)
  TimeInForce: Day

  Example:
    Ask = $1.56
    Limit = $1.56 × 0.95 = $1.482

    If filled: entry_premium = $1.482

STEP 2: Create Profit Target Order
  Type: LimitOrder (not StopLimit)
  Side: SELL
  Price: entry_premium × 1.50
  TimeInForce: GTC

  Example:
    entry_premium = $1.482
    target_price = $1.482 × 1.50 = $2.223

STEP 3: Create Stop Loss Order
  Type: StopLimitOrder (critical: StopLimit, not just Stop)
  Side: SELL
  Stop Price: entry_premium × (1 - stop_loss_pct)
  Limit Price: stop_price × 0.98 (2% slippage buffer)
  TimeInForce: GTC

  Example (25% stop):
    entry_premium = $1.482
    stop_price = $1.482 × 0.75 = $1.1115
    limit_price = $1.1115 × 0.98 = $1.089

STEP 4: Link as OCO Group
  Tag both orders with same OCO group ID
  QuantConnect automatically pairs them
  When one fills → other auto-cancels

STEP 5: Time Stop Override (3:45 PM)
  IF current_time >= 15:44:
    CancelOrder(profit_target_order)
    CancelOrder(stop_loss_order)
    PlaceOrder(MarketOrder to SELL all contracts)

STEP 6: Cooloff Period (Prevent Overtrading)
  After OCO resolves (either fills or time stop):
    next_eligible_entry_time = close_time + 30 minutes
    Don't trade options again until after this time
    Enforces max 1 trade per day with buffer
